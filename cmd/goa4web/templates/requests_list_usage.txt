Usage:
  {{.Prog}} requests list [flags]

List admin requests from the queue. Results are returned as JSON.

Flags:
  --status      Filter by status (pending or archived).
  --offset      Pagination offset (matches admin UI pagination semantics).
  --page-size   Page size (0 uses the configured default).

Examples:
  # List pending requests (default)
  {{.Prog}} requests list

  # List archived requests starting at offset 40
  {{.Prog}} requests list --status archived --offset 40 --page-size 20
List requests in the admin request queue. By default, only pending requests
are shown. Use -status archived to view historical requests.

Flags:
  -status  Filter by request status (pending, archived, accepted, rejected, query).
  -type    Filter by request change table.
  -json    Output results as machine-readable JSON.

Examples:
  {{.Prog}} requests list
  {{.Prog}} requests list -status archived
  {{.Prog}} requests list -status accepted -json
  {{.Prog}} requests list -type user_emails
