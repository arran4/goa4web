Usage:
  {{.Prog}} [flags] <command> [<args>]

This is the main entry point for the goa4web application. It provides a
command-line interface for managing the web server and its various
components.

{{template "flag_groups_section" .FlagGroups}}

Commands:
  help    Show help for a command.
  usage   Report usage statistics and summaries.
  serve   Run the web server. This is the primary command for running the
          application.
  user    Manage users. This includes adding, removing, and modifying user
          accounts.
  perm    Manage permissions. This allows you to control what actions users
          can perform.
  role    Manage roles. Roles are used to group users together and assign
          permissions to them.
  board   Manage image boards. This includes creating, deleting, and modifying
          image boards.
  blog    Manage blog entries. This allows you to create, edit, and delete
          blog posts.
  news    Manage news posts. This is used for creating and managing news
          articles.
  announcement Manage announcements. This includes listing, adding, and
          deleting announcements tied to news posts.
  faq     Manage frequently asked questions. This allows you to create and
          update a list of FAQs.
  forum   Manage forum topics and threads. This includes maintenance tasks for
          the public forums.
  private-forum Manage private forum topics and threads. This includes
          maintenance tasks for the private forums.
  writing Manage writing articles. This is used for managing articles and other
          written content.
  ipban   Manage IP bans. This allows you to block specific IP addresses from
          accessing the application.
  requests Manage admin requests. This is used for listing and inspecting
          administrative requests.
  imagebbs Manage ImageBBS moderation tasks.
  audit   Show recent audit log entries. This provides a log of all actions
          performed in the application.
  files   Manage uploaded image files. This includes listing and purging
          image board uploads.
  notifications Manage site notifications and task templates.
  db      Manage the database. This includes creating, migrating, and seeding
          the database.
  lang    Manage languages. This allows you to add and manage different
          languages for the application.
  maintenance Run one-off maintenance tasks that mirror admin actions.
  server  Manage the running server. This provides commands for interacting
          with the live server.
  password Manage pending password resets. This is used for managing password
          reset requests.
  email   Manage emails. This includes sending and managing email
          notifications.
  dlq     Manage dead letter queue entries.
  requests Manage admin request queue actions such as listing, accepting,
          rejecting, and commenting on requests.
  config  Manage configuration. This allows you to view and modify the
          application's configuration.
  test    Run manual tests. This is used for running specific tests that are
          not part of the automated test suite.

For example: {{.Prog}} board list -verbosity 1
