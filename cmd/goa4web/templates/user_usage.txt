Usage:
  {{.Prog}} user <command> [<args>]

The user command provides a suite of tools for managing user accounts. You can
add, modify, and remove users, as well as manage their roles, permissions, and
account status. This is the primary interface for all user-related
administrative tasks.

Commands:
  add         Add a new user to the system. Requires a username and password.
  add-admin   Add a new user with administrator rights. This is a convenient
              shortcut for creating a new admin account.
  make-admin  Grant administrator rights to an existing user.
  add-role    Grant a specific role to a user. Roles are used to assign a
              pre-defined set of permissions.
  remove-role Remove a role from a user, revoking the associated permissions.
  list-roles  List all available roles that can be assigned to users.
  list        List all users in the system, with options for filtering and
              sorting.
  deactivate  Deactivate a user's account, preventing them from logging in.
  activate    Reactivate a previously deactivated user account.
  approve     Approve a user account that is pending approval.
  reject      Reject a pending user account, with an optional reason.
  comment     Add an administrative comment for a user.
  comments    Manage administrative comments for a user.
  roles       List the roles assigned to a specific user.
  rename      Rename a user account, updating the username.
  profile     Display detailed profile information for a user.
  password    Manage a user's password.
  email       Manage a user's email addresses.
  unverified-emails List all unverified email addresses.
  subscriptions List a user's subscriptions.

Examples:
  # Add a new user named "bob" with the password "secret"
  {{.Prog}} user add -username bob -password secret

  # Deactivate the user "bob"
  {{.Prog}} user deactivate -username bob

  # List all users
  {{.Prog}} user list

  # Approve a user with ID 3
  {{.Prog}} user approve -id 3

  # Reject a user with ID 3 and provide a reason
  {{.Prog}} user reject -id 3 -reason "Spam account"

  # List all administrative comments for user with ID 3
  {{.Prog}} user comments list -id 3

  # Add an administrative comment for user with ID 3
  {{.Prog}} user comment add -id 3 -comment "needs review" -admin-id 1 --internal

  # List all roles assigned to the current user
  {{.Prog}} user roles

  # Rename a user from "alice" to "alice2"
  {{.Prog}} user rename -from alice -to alice2
